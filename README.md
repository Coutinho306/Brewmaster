# Brewmaster — Data Pipeline Intelligence Agent

Ask questions about your data pipelines in plain English. Brewmaster routes your query to the right specialist agent and returns a consolidated answer.

```
"How does the gold layer work?"               →  RAG Agent      (docs)
"Which pipelines failed most this month?"     →  DB Agent       (SQL)
"Is the Open Brewery DB API down?"            →  Web Agent      (live search)
"What is the difference between ETL and ELT?" →  Orchestrator   (direct answer)
```

Built on the [Bees-Brewery-Pipeline](https://github.com/Coutinho306/Bees-Brewery-Pipeline) project — a real medallion architecture (Bronze → Silver → Gold) using Apache Airflow.

## Quickstart

> `setup.sh` works on **Linux and WSL 2**. If you are on macOS or native Windows (Git Bash), skip to the [macOS tutorial](#macos-setup) or [Windows tutorial](#windows-setup) below.

### 1. Clone

```bash
git clone https://github.com/Coutinho306/brewmaster
cd Brewmaster
```

### 2. Run setup

```bash
./setup.sh
```

On first run the script creates `.env` from the template and exits, asking you to fill in your keys.

### 3. Fill in your API keys

Edit `.env`:

```env
OPENAI_API_KEY=sk-...        # LLM + embeddings (required)
TAVILY_API_KEY=tvly-...      # web search (required)
LANGSMITH_API_KEY=...        # tracing (optional)
```

Get your keys here:

- **OpenAI** (required) — [platform.openai.com](https://platform.openai.com/settings/proj_VDp59aHIqa3qpVQe7ih8MNlI/api-keys)
- **Tavily** (required) — [app.tavily.com](https://app.tavily.com/home)
- **Anthropic** (optional) — [console.anthropic.com](https://console.anthropic.com/settings/keys) · not used natively, swap model in `core/config.py`
- **Google Gemini** (optional) — [aistudio.google.com](https://aistudio.google.com/app/apikey) · not used natively, swap model in `core/config.py`

### 4. Start

```bash
./setup.sh
```

Installs dependencies, seeds the database, and starts both servers.

### 5. Open Brewmaster

- Chat UI → http://localhost:3000
- LangGraph API → http://localhost:2024

```bash
./stop.sh   # stop everything
```

## Architecture

```
User Query
      ↓
Orchestrator (gpt-5-nano, LangGraph)
  ↙       ↓        ↘          ↓
RAG       DB       Web     Direct
Agent    Agent    Agent    Answer
  ↓        ↓        ↓
Chroma  SQLite   Tavily
  ↘        ↓       ↙
      Final Answer
```

**Orchestrator** — routes every query; answers directly when no specialist is needed  
**RAG Agent** — searches pipeline docs in ChromaDB · architecture and "how does X work?" questions  
**DB Agent** — converts natural language to SQL, queries SQLite · counts, failures, rates, dates  
**Web Agent** — live search via Tavily · outages, external tools, real-time info

## Stack

**Agents** — LangGraph · LangChain v1  
**LLM** — OpenAI gpt-5-nano  
**Embeddings** — OpenAI text-embedding-3-small  
**Vector store** — ChromaDB  
**Database** — SQLite  
**Web search** — Tavily  
**UI** — Next.js (agent-chat-ui)

## Why fictional pipeline data?

The DB agent queries a SQLite database of pipeline run history. This data is intentionally fictional, generated by `data/seed_database.py` using [Faker](https://faker.readthedocs.io/).

The real Bees-Brewery-Pipeline Airflow instance typically has only a handful of runs (triggered manually during development), which would make most DB queries return *"not enough data"* — not a useful demo.

The fictional dataset solves this:

- **6 months of history** (Oct 2025 – Apr 2026) across **12 pipelines** spanning Bronze, Silver, Gold, and monitoring layers
- **~3,300 pipeline runs** capped at 60 runs/month per pipeline (2×/day for ingestion, 1×/day for transforms, weekly for reports) plus scattered manual reruns
- **~310 alerts** (failures, SLA breaches, anomalies — ~35 % resolved with owner and timestamp)
- **~15,000 data quality checks** covering check types per layer (min_record_count, unique_id, schema validation, etc.)

Pipeline names follow the real project's naming convention (`bees_ingestion_api_<layer>_<domain>`) and include:
`bees_ingestion_api_bronze_brewery_data`, `bees_ingestion_api_bronze_monitoring_data`, `bees_ingestion_api_bronze_sales_data`, `bees_ingestion_api_bronze_taproom_events`, `bees_ingestion_api_silver_brewery_transform`, `bees_ingestion_api_silver_sales_transform`, `bees_ingestion_api_gold_brewery_data`, `bees_ingestion_api_gold_sales_data`, `bees_ingestion_api_gold_taproom_analytics`, `bees_data_quality_monitor`, `bees_freshness_sla_check`, `bees_schema_drift_detector`.

Failure modes, error messages, and data quality check names mirror the real Bees project exactly — only the volume and history are synthetic. The RAG docs are fetched live from the real GitHub repository.

## Evaluation

```bash
# Run all
uv run pytest evals/ -v && uv run python evals/eval_rag.py

# Individual
uv run pytest evals/test_db_agent.py -v       # DB agent — SQL correctness + SELECT guard (~30s)
uv run pytest evals/test_orchestrator.py -v   # Orchestrator — routing + refusal rules (~30s)
uv run pytest evals/test_web_agent.py -v      # Web agent — search tool + response (~30s)
uv run python evals/eval_rag.py               # RAG agent — RAGAS metrics (~2-3min)
```

Results are saved to `evals/results/` as timestamped JSON files.

## macOS Setup

> `setup.sh` is Linux-only. Follow these steps manually on macOS.

### Prerequisites

```bash
# Install Homebrew if you don't have it
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install uv (Python package manager)
brew install uv

# Install Node.js + pnpm
brew install node
npm install -g pnpm
```

### Steps

**1. Clone the repo**

```bash
git clone https://github.com/Coutinho306/brewmaster
cd brewmaster
```

**2. Create your `.env`**

```bash
cp .env.example .env
```

Edit `.env` and fill in your API keys (`OPENAI_API_KEY`, `TAVILY_API_KEY`).

**3. Install Python dependencies**

```bash
uv sync
```

**4. Seed the database**

```bash
uv run python scripts/seed_db.py
```

**5. Install UI dependencies**

```bash
cd agent-chat-ui
pnpm install
cd ..
```

**6. Start the LangGraph server** (in one terminal)

```bash
uv run langgraph dev --no-browser
```

**7. Start the chat UI** (in a second terminal)

```bash
cd agent-chat-ui
pnpm dev
```

**8. Open Brewmaster**

- Chat UI → http://localhost:3000
- LangGraph API → http://localhost:2024

**To stop:** press `Ctrl+C` in both terminals.

`setup.sh` seeds the database and builds the vector store automatically on first run. Use these commands only to force a full reset:

```bash
uv run python scripts/seed_db.py   # reset SQLite database (12 pipelines, 6 months of history)
```
